🎯 2. 测试策略

2.1 测试方法

✓ 功能测试（Functional）
逐模块验证 PRD 定义的所有功能点：正向流程、异常分支、字段校验、边界值。
✓ 接口测试（API）
Pytest + httpx 验证 RESTful：状态码、请求/响应 Schema、Header、Token 鉴权、幂等性、错误码。
✓ UI 测试（E2E）
Playwright 模拟用户操作：登录、CRUD、拖拽、上传、视图切换；多浏览器覆盖。
✓ 性能测试（Performance）
Locust 100 QPS 并发；响应时间 P95/P99 监控；图片上传/AI 检索耗时统计。
✓ 安全测试（Security）
SQL 注入、XSS、CSRF、JWT 伪造、越权、文件上传漏洞、密码爆破、敏感信息泄露扫描。
✓ 兼容性测试（Compatibility）
Chrome/Edge/Firefox/Safari、屏幕分辨率、移动端、多数据库方言。

2.2 测试阶段 & 准入准出

HTTP 401；INVALID_TOKEN；记录审计日志
