# 最终最优落地方案（补齐图片存储 | OpenProject + RAGFlow + GraphRAG + MinIO 对象存储）
整体架构不变，**新增 MinIO 统一文件/图片存储层**，所有需求文档截图、架构图、数据库ER图、故障截图、上线附件图片统一托管，RAG 支持图片OCR解析入库，AI可检索图片内容。
> 选型结论：整套全部开源商用免费、私有化部署、3周MVP落地，不用从零开发后台、权限、用户、流程。

## 一、完整五层架构（新增图片存储层）
```
前端层：OpenProject改造前端 → 【版本变更管理】专属菜单
业务层：OpenProject二次开发（业务线/产品/版本/变更主逻辑）
附件&图片存储层：MinIO（所有图片、PDF、Excel、截图、架构图统一存对象存储）
AI-RAG层：
  ├ RAGFlow：文档解析 + 图片OCR + 向量入库、自然语言问答
  ├ GraphRAG：业务链路知识图谱、变更影响自动推理
数据采集层：Flyway(DB变更)+配置中心(配置变更)自动同步变更记录
数据库：MySQL（结构化业务数据）
```
### 各组件选型理由
1. **OpenProject(GPLv3)**：复用组织、业务线分组、权限、审批、用户体系，只新增变更模块；
2. **MinIO(Apache2.0)**：业界标准开源对象存储，专门存图片/附件，支持缩略图、预览、分片上传、权限鉴权、外链临时地址；**替代本地磁盘存储**，解决图片散乱、丢失、无法统一检索问题；
3. **RAGFlow(Apache2.0)**：自带图片OCR能力，上传的流程图、截图、ER图自动提取文字向量化，自然语言搜图：“找出所有结算表修改相关截图”；
4. **GraphRAG**：构建业务-数据表-接口图谱，自动计算变更影响范围；

## 二、图片&附件全链路落地规则
### 1、MinIO存储目录规范（按业务线自动分桶，方便运维）
```
桶名格式：biz-{业务线编码}
├ /version/{版本ID}/docs  需求文档、上线方案
├ /version/{版本ID}/img   架构图、BUG截图、数据库截图、流程图、原型图
├ /version/{版本ID}/report 测试报告、复盘文档
```
- OpenProject上传图片时，后端自动生成MinIO路径，数据库只存**图片URL+MinIO对象路径**，不存二进制；
- 预览直接返回MinIO临时签名链接，支持前端在线预览图片/PDF。

### 2、三类图片接入场景
1. **变更附件图片**：每条变更条目可上传多张图片（BUG截图、修改前后对比图、SQL执行截图），上传后自动推送RAGFlow做OCR识别，文本存入向量库；
2. **版本文档插图**：版本说明里嵌入架构图、ER图，RAG解析图片文字，支持按图片内容检索变更；
3. **故障复盘图片**：线上问题截图、日志截图，入库RAG，后续同类变更AI自动带出历史故障图片与解决方案。

### 3、RAG图片联动逻辑
1. 用户上传图片→MinIO存文件→回调RAGFlow；
2. RAG自动OCR提取图片文字+多模态向量；
3. 绑定所属【业务线+版本+变更ID】存入知识库；
4. 自然语言查询示例：
> “医保结算相关字段改动所有截图”→AI召回对应版本+关联图片+变更记录。

## 三、OpenProject改造细节（业务分层映射不变+新增图片字段）
|系统层级|OpenProject原生载体|扩展内容（含图片）|
|----|----|----|
|业务线|项目集(Portfolio)|负责人、上下游业务、存储桶编码（绑定MinIO桶）|
|产品|项目(Project)|环境信息、DB清单、默认图片存储路径前缀|
|迭代版本|里程碑(Milestone)|版本类型、上线时间、回滚方案、版本图集入口|
|变更明细(自研模块)|里程碑子页面【变更清单】|变更5大类+**图片附件列表（存MinIO地址数组）**|

### 变更条目新增字段
- `img_urls`：JSON数组，存储MinIO图片签名地址，前端直接预览；
- `file_ref`：关联文档/PDF地址。

## 五、MVP开发排期（3周落地可用系统）
### 第1周：环境部署
1. 部署OpenProject+MySQL；
2. 部署MinIO，配置桶策略、API密钥；
3. 部署RAGFlow、GraphRAG服务，打通API连通性。

### 第2周：OpenProject定制开发
1. 配置业务线(项目集)、产品(项目)、版本(里程碑)自定义字段；
2. 开发【变更清单】子页面，接入MinIO上传，图片预览；
3. 开发变更数据自动推送RAGFlow接口。

### 第3周：AI联动+调试
1. GraphRAG导入业务依赖，构建业务/库/表/接口知识图谱；
2. 调试图片OCR入库、自然语言查询、AI自动推荐影响范围；
3. 对接Flyway，实现DB变更自动同步至变更清单。
